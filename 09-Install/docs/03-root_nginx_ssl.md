# Step 0d – 03-root_nginx_ssl.sh

**Script:** `09-Install/03-root_nginx_ssl.sh`
**Runs as:** `root` or `oracle` with sudo
**Phase:** 0 – OS Preparation

---

## Purpose

Configure SSL/TLS in Nginx: bind the customer-provided certificate, set secure TLS
parameters, validate the configuration, and start Nginx. This is the final step before
WebLogic is reachable from the outside.

---

## Without the Script (manual)

### 1. Copy certificate files

Place the SSL certificate and key in a secure location:

```bash
mkdir -p /etc/nginx/ssl
cp your-server.crt    /etc/nginx/ssl/server.crt
cp your-server.key    /etc/nginx/ssl/server.key
cp intermediate-ca.crt /etc/nginx/ssl/chain.crt   # if applicable
chmod 600 /etc/nginx/ssl/server.key
chmod 644 /etc/nginx/ssl/server.crt /etc/nginx/ssl/chain.crt
```

### 2. Add SSL block to Nginx config

Add to the `server { listen 443 ssl; ... }` block in `/etc/nginx/conf.d/oracle-fmw.conf`:

```nginx
    # Certificate
    ssl_certificate     /etc/nginx/ssl/server.crt;
    ssl_certificate_key /etc/nginx/ssl/server.key;

    # TLS versions (1.2 + 1.3 only)
    ssl_protocols TLSv1.2 TLSv1.3;

    # Cipher suite (ECDHE only, no RC4/3DES/EXPORT)
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:DHE-RSA-AES128-GCM-SHA256';
    ssl_prefer_server_ciphers on;

    # Session cache
    ssl_session_cache   shared:SSL:10m;
    ssl_session_timeout 10m;

    # HSTS (enable after verifying SSL works correctly)
    # add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
```

### 3. Validate and reload

```bash
nginx -t
systemctl reload nginx
# or start if not yet running:
systemctl start nginx
systemctl status nginx
```

### 4. Verify SSL

```bash
# Check certificate details:
openssl s_client -connect localhost:443 -showcerts </dev/null 2>/dev/null \
    | openssl x509 -noout -text | grep -E "Subject:|Issuer:|Not After"

# Or from the outside:
curl -v https://your-server-fqdn/forms/frmservlet
```

---

## What the Script Does

- Reads `SSL_CERT_FILE`, `SSL_KEY_FILE`, `SSL_CHAIN_FILE` from `environment.conf`
  (or prompts if not set)
- Validates certificate file: exists, readable, not expired, matches key (modulus check)
- Copies cert/key to `/etc/nginx/ssl/` with correct permissions
- Injects SSL directives into the existing Nginx config generated by `02-root_nginx.sh`
- Runs `nginx -t` before applying
- Starts or reloads Nginx
- Runs a quick SSL handshake test via `openssl s_client`
- Reports certificate subject, issuer, and expiry date

---

## environment.conf Parameters

| Parameter | Example | Description |
|---|---|---|
| `SSL_CERT_FILE` | `/srv/certs/server.crt` | Path to PEM certificate |
| `SSL_KEY_FILE` | `/srv/certs/server.key` | Path to private key |
| `SSL_CHAIN_FILE` | `/srv/certs/chain.crt` | Intermediate CA chain (optional) |

---

## Flags

| Flag | Description |
|---|---|
| (none) | Validate cert files and show planned config changes |
| `--apply` | Write SSL config, copy cert files, reload Nginx |
| `--help` | Show usage |

## Verification

```bash
# Certificate expiry
openssl x509 -noout -enddate -in /etc/nginx/ssl/server.crt

# Key matches cert (moduli must be identical)
openssl x509 -noout -modulus -in /etc/nginx/ssl/server.crt | md5sum
openssl rsa  -noout -modulus -in /etc/nginx/ssl/server.key | md5sum

# Live TLS check
openssl s_client -connect localhost:443 </dev/null 2>/dev/null \
    | grep -E "^subject|^issuer|Verify return code"

# Protocol check (should NOT show SSLv3/TLSv1.0/TLSv1.1)
nmap --script ssl-enum-ciphers -p 443 localhost
```

---

## Notes

- This script expects `02-root_nginx.sh` to have already created the Nginx config
- In production: always use a CA-signed certificate (not self-signed)
- HSTS header is commented out by default — enable after confirming SSL works correctly
- For certificate renewal: re-run with `--apply` after replacing cert files in their source location
