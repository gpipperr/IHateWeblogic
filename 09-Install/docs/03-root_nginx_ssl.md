# Step 0d – 03-root_nginx_ssl.sh

**Script:** `09-Install/03-root_nginx_ssl.sh`
**Runs as:** `root` or `oracle` with sudo
**Phase:** 0 – OS Preparation

---

## Purpose

Configure SSL/TLS in Nginx: bind the customer-provided certificate, set secure TLS
parameters, validate the configuration, and start Nginx. This is the final step before
WebLogic is reachable from the outside.

The base Nginx config (upstream blocks + location blocks) is generated by `02-root_nginx.sh`.
This script adds only the SSL directives to the existing config.

---

## Architecture

NGINX is the single SSL termination point in front of WebLogic. All external HTTPS
traffic is handled by NGINX; WebLogic listens on `localhost` only and never handles SSL.

```
Client (HTTPS :443)
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│  NGINX  –  SSL Termination + Reverse Proxy                  │
│  /etc/nginx/conf.d/oracle-fmw.conf                          │
│                                                             │
│  Port 80  →  301 redirect → https://                        │
│  Port 443 →  ssl_certificate / ssl_certificate_key          │
└─────────────────────────────────────────────────────────────┘
      │  HTTP plain — loopback only (127.0.0.1)
      │
      ├─── /forms/*    →  http://127.0.0.1:9001  (WLS_FORMS)
      ├─── /reports/*  →  http://127.0.0.1:9002  (WLS_REPORTS)
      └─── /console    →  http://127.0.0.1:7001  (Admin Server)
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│  WebLogic  (WLS_LISTEN_ADDRESS = localhost)                  │
│  Admin Server   127.0.0.1:7001                              │
│  WLS_FORMS  MS  127.0.0.1:9001                              │
│  WLS_REPORTS MS 127.0.0.1:9002                              │
└─────────────────────────────────────────────────────────────┘
```

**Consequences for the WebLogic configuration:**

- `WLS_LISTEN_ADDRESS=localhost` is set during setup interview and applied at domain
  creation — WebLogic is not reachable from outside the host at all
- No SSL keystore, no certificate management inside WebLogic
- Admin Console is reachable via `https://server-fqdn/console` through NGINX,
  or directly via `http://localhost:7001/console` with an SSH tunnel
- WebLogic must know the external URL to generate correct redirects —
  see [WebLogic Frontend Host Configuration](#weblogic-frontend-host-configuration) below

**Why not SSL inside WebLogic directly?**

- One certificate location (NGINX only) — not two
- No `java.security.KeyStore` management, no expired-WLS-keystore incidents
- NGINX handles HTTP/2, HSTS, and cipher policy centrally
- Forms and Reports traffic is unencrypted only on loopback (127.0.0.1) —
  it never leaves the host

## Nginx Proxy Configuration

The base configuration is generated by `02-root_nginx.sh`.
This script (`03-root_nginx_ssl.sh`) injects only the SSL directives.

### Full structure: /etc/nginx/conf.d/oracle-fmw.conf

```nginx
# --- Upstream blocks ----------------------------------------------------------
upstream wls_admin {
    server 127.0.0.1:7001;
    keepalive 8;
}

upstream wls_forms {
    server 127.0.0.1:9001;
    keepalive 16;
}

upstream wls_reports {
    server 127.0.0.1:9002;
    keepalive 8;
}

# --- HTTP → HTTPS redirect ---------------------------------------------------
server {
    listen 80;
    server_name wls01.company.local;
    return 301 https://$host$request_uri;
}

# --- HTTPS server block -------------------------------------------------------
server {
    listen 443 ssl;
    server_name wls01.company.local;

    # --- SSL directives (injected by 03-root_nginx_ssl.sh) --------------------
    ssl_certificate     /etc/nginx/ssl/server.crt;
    ssl_certificate_key /etc/nginx/ssl/server.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:DHE-RSA-AES128-GCM-SHA256';
    ssl_prefer_server_ciphers on;
    ssl_session_cache   shared:SSL:10m;
    ssl_session_timeout 10m;
    # add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    # --- Proxy headers (global for all location blocks) -----------------------
    proxy_set_header Host              $host;
    proxy_set_header X-Real-IP         $remote_addr;
    proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto https;
    proxy_http_version 1.1;
    proxy_set_header Connection "";    # enable keepalive to upstream

    # --- Forms ----------------------------------------------------------------
    location /forms/ {
        proxy_pass            http://wls_forms;
        proxy_read_timeout    300s;
        proxy_connect_timeout  60s;
        proxy_send_timeout    300s;
        proxy_buffering       off;     # Forms applets must stream – buffering causes timeouts
        proxy_max_temp_file_size 0;
    }

    location /frmservlet {
        proxy_pass     http://wls_forms/forms/frmservlet;
        proxy_buffering off;
    }

    # --- Reports --------------------------------------------------------------
    location /reports/ {
        proxy_pass            http://wls_reports;
        proxy_read_timeout    600s;    # Reports jobs can run several minutes
        proxy_connect_timeout  60s;
        proxy_send_timeout    600s;
        proxy_buffering       on;
        proxy_buffer_size     128k;
        proxy_buffers         4 256k;
        proxy_busy_buffers_size 256k;
    }

    # --- WebLogic Admin Console -----------------------------------------------
    # Restrict to management network or use SSH tunnel – never expose publicly.
    location /console {
        # allow 10.0.0.0/8;
        # deny all;
        proxy_pass            http://wls_admin;
        proxy_read_timeout    120s;
        proxy_connect_timeout  30s;
    }

    # --- Health check (optional) ----------------------------------------------
    location /nginx_status {
        stub_status;
        allow 127.0.0.1;
        deny all;
    }
}
```

### Key proxy settings explained

| Setting | Value | Reason |
|---|---|---|
| `proxy_buffering off` (Forms) | off | Forms applets must stream to the client; buffering causes download timeouts |
| `proxy_read_timeout` (Reports) | 600 s | Reports jobs can run several minutes; default 60 s would terminate them |
| `X-Forwarded-Proto: https` | always set | WebLogic checks this header to generate correct HTTPS URLs in redirects |
| `proxy_http_version 1.1` | 1.1 | Required for keepalive connections to upstream |
| `keepalive` (upstream) | 8–16 | Connection reuse reduces overhead for frequent Forms/Reports requests |

## WebLogic Frontend Host Configuration

When WebLogic generates absolute URLs (redirects, admin console links, Reports
output file references), it uses its own listen address and port — which is
`localhost:7001/9001/9002`. Without correction, these redirects would break because
the browser cannot reach `localhost` from the outside.

The fix is to set **Frontend Host** and **Frontend HTTPS Port** on each WebLogic
server. This is done automatically during domain creation via WLST (the domain
creation script reads `WLS_LISTEN_ADDRESS` and the server FQDN from `environment.conf`).

### Manual configuration via WLST (if not set during domain creation)

```python
connect('webadmin', '<password>', 't3://localhost:7001')
edit()
startEdit()

# Admin Server
cd('/Servers/AdminServer/WebServer/AdminServer')
set('FrontendHost', 'wls01.company.local')
set('FrontendHTTPSPort', 443)
set('FrontendHTTPPort', 80)

# Managed Servers
for srv in ['WLS_FORMS', 'WLS_REPORTS']:
    cd('/Servers/' + srv + '/WebServer/' + srv)
    set('FrontendHost', 'wls01.company.local')
    set('FrontendHTTPSPort', 443)
    set('FrontendHTTPPort', 80)

save()
activate()
disconnect()
```

### Manual configuration via Admin Console

1. Log in: `https://server-fqdn/console` (or via SSH tunnel: `http://localhost:7001/console`)
2. **Domain structure → Environment → Servers → `<server-name>` → Protocols → HTTP**
3. Set **Frontend Host**, **Frontend HTTP Port** (80), **Frontend HTTPS Port** (443)
4. Repeat for each Managed Server
5. Activate changes → rolling restart required

## Without the Script (manual)

### 1. Copy certificate files

Place the SSL certificate and key in a secure location:

```bash
mkdir -p /etc/nginx/ssl
cp your-server.crt    /etc/nginx/ssl/server.crt
cp your-server.key    /etc/nginx/ssl/server.key
cp intermediate-ca.crt /etc/nginx/ssl/chain.crt   # if applicable
chmod 600 /etc/nginx/ssl/server.key
chmod 644 /etc/nginx/ssl/server.crt /etc/nginx/ssl/chain.crt
```

### 2. Add SSL block to Nginx config

Add to the `server { listen 443 ssl; ... }` block in `/etc/nginx/conf.d/oracle-fmw.conf`:

```nginx
    # Certificate
    ssl_certificate     /etc/nginx/ssl/server.crt;
    ssl_certificate_key /etc/nginx/ssl/server.key;

    # TLS versions (1.2 + 1.3 only)
    ssl_protocols TLSv1.2 TLSv1.3;

    # Cipher suite (ECDHE only, no RC4/3DES/EXPORT)
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:DHE-RSA-AES128-GCM-SHA256';
    ssl_prefer_server_ciphers on;

    # Session cache
    ssl_session_cache   shared:SSL:10m;
    ssl_session_timeout 10m;

    # HSTS (enable after verifying SSL works correctly)
    # add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
```

### 3. Validate and reload

```bash
nginx -t
systemctl reload nginx
# or start if not yet running:
systemctl start nginx
systemctl status nginx
```

### 4. Verify SSL

```bash
# Check certificate details:
openssl s_client -connect localhost:443 -showcerts </dev/null 2>/dev/null \
    | openssl x509 -noout -text | grep -E "Subject:|Issuer:|Not After"

# Or from the outside:
curl -v https://your-server-fqdn/forms/frmservlet
```

## What the Script Does

- Reads `SSL_CERT_FILE`, `SSL_KEY_FILE`, `SSL_CHAIN_FILE` from `environment.conf`
  (or prompts if not set)
- Validates certificate file: exists, readable, not expired, matches key (modulus check)
- Copies cert/key to `/etc/nginx/ssl/` with correct permissions
- Injects SSL directives into the existing Nginx config generated by `02-root_nginx.sh`
- Runs `nginx -t` before applying
- Starts or reloads Nginx
- Runs a quick SSL handshake test via `openssl s_client`
- Reports certificate subject, issuer, and expiry date

---

## environment.conf Parameters

| Parameter | Example | Description |
|---|---|---|
| `SSL_CERT_FILE` | `/srv/certs/server.crt` | Path to PEM certificate |
| `SSL_KEY_FILE` | `/srv/certs/server.key` | Path to private key |
| `SSL_CHAIN_FILE` | `/srv/certs/chain.crt` | Intermediate CA chain (optional) |
| `WLS_LISTEN_ADDRESS` | `localhost` | WebLogic bind address — `localhost` when NGINX is used |
| `WLS_ADMIN_PORT` | `7001` | Admin Server port (upstream target) |
| `WLS_FORMS_PORT` | `9001` | Forms Managed Server port (upstream target) |
| `WLS_REPORTS_PORT` | `9002` | Reports Managed Server port (upstream target) |

## Verification

```bash
# Certificate expiry
openssl x509 -noout -enddate -in /etc/nginx/ssl/server.crt

# Key matches cert (moduli must be identical)
openssl x509 -noout -modulus -in /etc/nginx/ssl/server.crt | md5sum
openssl rsa  -noout -modulus -in /etc/nginx/ssl/server.key | md5sum

# Live TLS check
openssl s_client -connect localhost:443 </dev/null 2>/dev/null \
    | grep -E "^subject|^issuer|Verify return code"

# Protocol check (should NOT show SSLv3/TLSv1.0/TLSv1.1)
nmap --script ssl-enum-ciphers -p 443 localhost

# End-to-end proxy test
curl -sk https://localhost/forms/frmservlet | head -5
curl -sk https://localhost/reports/rwservlet/getserverinfo?statusformat=XML | head -10
```

## Notes

- This script expects `02-root_nginx.sh` to have already created the base Nginx config
  with the upstream and location blocks
- HSTS header is commented out by default — enable only after confirming SSL works
  correctly; once sent, HSTS cannot easily be revoked
- For certificate renewal: replace cert files in their source location, then re-run
  `03-root_nginx_ssl.sh --apply`
- The Admin Console location block has `allow`/`deny` commented out — restrict by IP
  in production environments or rely exclusively on SSH tunnel access
- WebLogic Frontend Host must be set after domain creation so that server-generated
  redirects use the external HTTPS URL, not `localhost`

## Certificate Options

### Option A – Company / Public CA Certificate (production)

The customer provides a CA-signed certificate.
Place the files in a secure staging location, then set in `environment.conf`:

```bash
SSL_CERT_FILE=/srv/certs/fullchain.pem   # cert + intermediate chain
SSL_KEY_FILE=/srv/certs/privkey.pem
```

`03-root_nginx_ssl.sh --apply` validates, copies, and deploys.

---

### Option B – Internal CA with Easy-RSA (internal/lab environments)

For environments without a public CA, Easy-RSA provides a simple way to operate
an internal CA and issue server certificates with correct SANs.
All certificates will be trusted by browsers/clients once the CA root is imported.

Reference: https://www.pipperr.de/dokuwiki/doku.php?id=linux:ca_on_oracle_linux_9

#### 1. Install Easy-RSA

```bash
dnf install oracle-epel-release-el9
dnf install easy-rsa openssl
```

#### 2. Initialize CA directory

```bash
mkdir -p /srv/gpi_ca
ln -s /usr/share/easy-rsa/3 /srv/gpi_ca/easy-rsa
chmod 700 /srv/gpi_ca

cd /srv/gpi_ca
./easy-rsa/easyrsa init-pki
```

#### 3. Configure CA (edit `pki/vars`)

```
set_var EASYRSA_ALGO         ec           # elliptic curve (modern, smaller keys)
set_var EASYRSA_CURVE        secp384r1
set_var EASYRSA_CA_EXPIRE    3650         # CA valid 10 years
set_var EASYRSA_CERT_EXPIRE  825          # server certs valid ~2 years
set_var EASYRSA_CRL_DAYS     180
set_var EASYRSA_AUTO_SAN     1            # auto SAN from CN (required for modern browsers)
```

#### 4. Create the CA

```bash
./easy-rsa/easyrsa build-ca
# Prompts: CA passphrase + Common Name (e.g. "Company.local CA")
```

CA certificate created at: `pki/ca.crt` — this is the root to distribute to clients.

#### 5. Create server certificate

```bash
./easy-rsa/easyrsa build-server-full wls01.company.local nopass
```

Output files:
| File | Use |
|---|---|
| `pki/issued/wls01.company.local.crt` | Server certificate → `SSL_CERT_FILE` |
| `pki/private/wls01.company.local.key` | Private key → `SSL_KEY_FILE` |
| `pki/ca.crt` | CA root → distribute to all clients as `SSL_CHAIN_FILE` |

#### 6. Verify SAN is present

```bash
openssl x509 -in pki/issued/wls01.company.local.crt -noout -text \
    | grep -A2 "Subject Alternative Name"
# Expected: DNS:wls01.company.local
```

Modern browsers require SAN — a certificate without SAN will show a security error
even if the CN matches.

#### 7. Distribute CA root to clients

```bash
# On each client machine (Oracle Linux):
cp pki/ca.crt /etc/pki/ca-trust/source/anchors/company-local-ca.crt
update-ca-trust

# On Windows clients: import pki/ca.crt into Trusted Root Certification Authorities
# via certmgr.msc or Group Policy
```

#### 8. Set in environment.conf

```bash
SSL_CERT_FILE=/srv/gpi_ca/pki/issued/wls01.company.local.crt
SSL_KEY_FILE=/srv/gpi_ca/pki/private/wls01.company.local.key
SSL_CHAIN_FILE=/srv/gpi_ca/pki/ca.crt
```

Then run: `./09-Install/03-root_nginx_ssl.sh --apply`

#### Backup

> Loss of CA private key (`pki/private/ca.key`) means all issued certificates
> become unverifiable — back up `/srv/gpi_ca/` regularly to offline storage.

### Option C – Self-signed certificate (testing only)

Quick generation for smoke testing. Browsers will show a security warning.

```bash
openssl req -x509 -newkey rsa:4096 -sha256 -days 365 -nodes \
    -keyout /srv/certs/privkey.pem \
    -out    /srv/certs/fullchain.pem \
    -subj   "/CN=wls01.company.local" \
    -addext "subjectAltName=DNS:wls01.company.local"
```
